## Summary
- 将卡牌抽取范围从 4-5 星扩展到 **3-5 星**（`fetch_all_cards` 现在纳入 rarity=3 的卡牌）
- 新增 `parse_rarity_range()` 函数，支持**范围输入**方式（如 `3-5`、`4-5`、`3`），替代原有的枚举格式（`4`/`5`/`45`），降低记忆难度
- 交互模式（`draw_filtered`）和 CLI 快速模式（`fast_mode`）均已适配新的范围输入
- CLI `-r` 参数默认值从 `45` 改为 `3-5`
- 添加 Windows 控制台 UTF-8 编码支持，防止中文乱码
- 更新所有界面文案（`四星/五星` → `三~五星`）
- skill 副本已同步，README 已更新

## Tests
- 本项目无自动化测试套件，已通过代码审查确认：
  - `parse_rarity_range` 逻辑覆盖合法输入（`3-5`、`4-5`、`3`、`4`、`5`、空输入默认值）和非法输入（返回 None）
  - `build_available_pool` 的范围过滤使用 `>=`/`<=` 边界，与单值逻辑等效
  - 所有调用点参数签名已对齐

## Risks / Notes
- API 返回数据中三星卡牌数量较大，首次加载可能变慢
- 如 Bestdori API 的 `rarity` 字段存在非标准值（如 1、2 星），当前逻辑仅过滤 3/4/5，不会被纳入卡池
